# Developer Efficiency Analytics (DEA)

Система оценки справедливой стоимости работ и эффективной ставки разработчиков по проекту.

## Требования

- Node.js 18+
- PostgreSQL 14+
- pnpm (рекомендуется) или npm

## Быстрый старт

1. **Клонируйте и установите зависимости**

```bash
cd prototype
pnpm install
```

2. **Настройте переменные окружения**

```bash
cp .env.example .env.local
# Отредактируйте .env.local и укажите DATABASE_URL
```

3. **Создайте базу данных**

```bash
# Создайте БД в PostgreSQL
createdb dea_db

# Примените миграции
pnpm db:push

# Заполните справочники
pnpm db:seed
```

4. **Запустите dev-сервер**

```bash
pnpm dev
```

Откройте [http://localhost:3000](http://localhost:3000)

## Формулы расчета

### Эффективная ставка

```
Эффективная ставка = (Продуктивность × 100% / (3.8 × % Доля)) × Базовая ставка
```

Где:
- **Продуктивность** - средняя продуктивность по фактическим дням
- **3.8** - норма затрат времени на разработку в день
- **% Доля** - процент участия в роли разработчика

### Отклонение

```
Отклонение = (Эффективная ставка - Базовая ставка) / Базовая ставка × 100%
```

### Оценка стоимости

```
Стоимость = Эффективная ставка × Фактически отработанных дней
```

## Цветовая шкала отклонений

| Отклонение | Цвет | Интерпретация |
|------------|------|---------------|
| >= +10% | Зеленый | Перевыполнение |
| 0% - +10% | Салатовый | Хороший результат |
| -10% - 0% | Желтый | Норма |
| -20% - -10% | Оранжевый | Внимание |
| < -20% | Красный | Недовыполнение |
